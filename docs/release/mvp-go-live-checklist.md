# MVP 上线检查清单（Task 11）

## 发布前检查项

- [ ] API 基础回归通过：`health` 与 `metrics` 路由可用。
- [ ] 监控看板已配置并可查看 `availability`、`latency`、`error_rate`。
- [ ] 告警通道已验证（值班群或通知系统可收到告警）。
- [ ] 发布窗口已确认，已通知相关负责人（研发、测试、产品、运营）。
- [ ] 回滚版本已准备（上一个稳定镜像/构建产物可直接恢复）。

## 灰度发布步骤

1. 发布到灰度环境，先开放 10% 流量。
2. 观察 15 分钟，确认：
   - `availability >= 99.5%`
   - `p95 latency <= 500ms`
   - `error_rate <= 1%`
3. 若指标稳定，逐步扩大到 30% -> 50% -> 100%，每步至少观察 15 分钟。
4. 每次扩容前记录当前指标快照，出现异常立即停止扩容。

## 回滚策略

1. 触发条件（任一满足）：
   - 连续 5 分钟 `availability < 99%`
   - 连续 5 分钟 `error_rate > 2%`
   - `p95 latency > 1000ms` 且持续 10 分钟
2. 操作步骤：
   - 立即停止流量扩容并切回上一个稳定版本。
   - 验证 `health` 与核心用户路径可用。
   - 继续观察 15 分钟，确认指标恢复。
3. 回滚后处置：
   - 记录故障时间线与影响范围。
   - 输出根因与修复计划后再安排下一次灰度。
